# Component Library

Use only `report-*` classes for generated report HTML.

## Core Layout

- `report-page`: root report fragment.
- `report-card`: standard white report section.
- `report-cover`: reader-level cover container.
- `report-grid`: generic grid layout.
- `report-grid-2`: two-column grid that collapses on mobile.
- `report-divider`: subtle horizontal divider.

## Typography

- `report-title`: main reader title.
- `report-subtitle`: reader summary.
- `report-section-title`: section heading.
- `report-body`: readable content container.
- `report-meta`: metadata row.

## Status And Emphasis

- `report-badge`: neutral badge, used sparingly.
- `report-badge-success`: available for compatibility, rendered in the black-and-white system.
- `report-badge-warning`: available for compatibility, rendered in the black-and-white system.
- `report-badge-danger`: available for compatibility, rendered in the black-and-white system.
- `report-callout`: primary callout.
- `report-callout-warning`: warning callout.
- `report-callout-danger`: danger callout.
- `report-note`: quiet supporting note.

## Data Display

- `report-metric-grid`: metric card grid.
- `report-metric-card`: individual metric block.
- `report-table`: readable table.
- `report-timeline`: ordered next steps or timeline list.
- `report-download-bar`: Reader-owned fixed download menu container; do not include it in generated report HTML.

## Rich Static Components

- `report-stat-row`: compact row of static stats.
- `report-stat`: individual stat item inside `report-stat-row`.
- `report-quote`: emphasized static quotation or thesis block.
- `report-source-list`: list of source notes or references.
- `report-comparison-grid`: two-column comparison layout.
- `report-comparison-card`: individual comparison block.
- `report-progress`: static progress component.
- `report-progress-item`: one progress row.
- `report-progress-label`: label row for a progress item.
- `report-progress-track`: progress rail.
- `report-progress-bar`: filled progress indicator.
- `report-definition-list`: key-value terminology list.
- `report-risk-matrix`: static risk matrix container.
- `report-risk-row`: one row inside a risk matrix.
- `report-footnotes`: compact footnote list.

## Default Quiet Memo Sections

- `context`: why the report exists.
- `main-reading`: primary narrative.
- `interpretation`: what the observations mean.
- `supporting-evidence`: table or compact evidence.
- `reading-notes`: caveats, assumptions, or source notes.
- `closing`: read-only next steps.
