# Report Contract

This contract defines the HTML accepted by Personal HTML Report OS.

## Allowed

- Start with `<div class="report-page" data-report-type="...">`.
- Use stable `<section class="report-card" data-section-id="...">` blocks.
- Each meaningful section should include one `.report-section-title`; the frontend uses `data-section-id` and `.report-section-title` to generate the Table of Contents.
- Use only classes from the `report-*` design system.
- Use semantic static HTML such as `p`, `strong`, `table`, `thead`, `tbody`, `tr`, `th`, `td`, `ol`, and `li`.
- Use read-only action lists or next steps when actions are needed.
- Prefer the quiet memo section rhythm: `context`, `main-reading`, `interpretation`, `supporting-evidence`, `reading-notes`, and `closing`.

## Forbidden

- No `<html>`, `<head>`, or `<body>` tags.
- No `<script>` tags.
- No `<style>` tags.
- No inline `style` attributes.
- No external CSS or JavaScript.
- No interactive checkboxes or frontend state controls in Version 0.1.
- No create, edit, delete, search, filter, or admin dashboard controls inside report HTML.
- No custom Table of Contents inside report HTML; TOC is generated by the Report Reader.
- No download controls inside report HTML; the Reader owns the fixed top-right Download menu.
- No custom color systems. The default report body is black-and-white.
- Rich components are allowed only when they use approved `report-*` classes and remain static.

## Reader-Owned UI

- Breadcrumb navigation is fixed in the top-left screen area.
- Download is fixed in the top-right screen area.
- Table of Contents is generated by the Reader from report sections and displayed outside the report HTML.
- Report HTML is content only; it does not own navigation, downloads, TOC, editing controls, or page chrome.

## Phase Discipline

Phase 1 is static visual prototyping only. Storage, publish APIs, downloads, permissions, and report management workflows must be added in later phases.

## Local Validation

Validate generated report HTML before storage:

```bash
npm run validate:report -- path/to/report.html
```
